Quem Somos?

Monitorando Amazonas é uma aplicação web para monitoramento de território utilizado para o agropastoris. Através de gráficos, a responsabilidade da aplicação é mostrar toda a utilização do território das fazendas cadastradas na prefeitura de Xapuri. Com a disponibilização da aplicação, será possivel ver o cruzamento de dados, tais como:

Total de hectares;

Total de hectares desmatados;

Total de hectares reflorestados;

Total de hectares utilizados para plantio;

Nós da Monitorando Amazonas temos a total certeza, de que, com o passar dos anos, nossa aplicação ajudará muito no combate do desmatamento ilegal. Novas ideias estão por vir, não fique de fora dessa luta.
Nossa Equipe

Dois Engenheiros agrônomos - Capacitados para fazer todo o levantamento de campo, e coletar os dados para serem transformados em informações.

Um DBA - Responsável por todo o banco de dados que fará integração com a aplicação;

Dois desenvolvedores BACK-END - Responsáveis por criarem todas api's para alimentar a aplicação;

Um desenvolvedor FRONT-END - Responsável por criar nossa aplicação web;

Um QA - Responsável por todos os testes de qualidade da aplicação;
Nosso Produto

O intuito da nossa aplicação, é auxiliar os profissionais responsáveis por auditar, monitorar, fiscalizar e vistoriar fazendas do segmento de agropastoris do estado do Amazonas. Disponibilizando gráficos para o cruzamento de dados, para facilitar os fiscais das prefeituras credenciadas, a encontrarem divergências, falta de responsabilidade, agressão ao meio ambiente e etc.
Arquitetura

FRONT-END - Angular 11, comunicando com uma aplicação webApi.

BACK-END - Aplicação webApi em Java utilizando o FrameWork SpringBoot, cominucando com uma base de dados em mongoDB.

DATABASE - Banco de dados usando MongoDB. A utilização do mesmo, reduz custo na infraestrutura do projeto e acarreta melhor performace por seu schema em JSON.

DEPLOY - Para expor nossas aplicações, usaremos o Heroku, é uma plataforma cloud gratuita que já faz integração com o GITHUB. Reduz o custo na infraestrutura do projeto, e é eficiente para o momento do projeto.
